# World Objects & Props

Interactive and environmental world objects that don't fit a more specific category -- props, pickups, hazards, level-specific set pieces. If a module represents a physical thing placed in the world and isn't a vehicle or a support/mission system, it's probably here.

## Modules in this category

- **[Alarm](alarm)** — The `Alarm` module manages the activation and deactivation of alarms for occupied buildings within a 100-meter radius.
- **[Barbell](barbell)** — The `Barbell` module provides a utility function to detach the barbell from an object that is holding it.
- **[Beacon](beacon)** — The `Beacon` module represents a beacon object in the game world.
- **[Bench](bench)** — The `Bench` module provides utility functions for querying repair actions and handling object interactions.
- **[Binoculars](binoculars)** — The `Binoculars` module is a utility script that handles the functionality of binoculars in the game.
- **[Collectable](collectable)** — The `Collectable` module represents collectible objects in the game world.
- **[Crate](crate)** — The `Crate` module represents a winchable crate object in the game world.
- **[Danceradio](danceradio)** — The `Danceradio` module is a lifestyle prop that allows players to perform a dance animation.
- **[DangerousBuilding](dangerousbuilding)** — The `DangerousBuilding` module manages dangerous buildings in the game.
- **[Despawner](despawner)** — The `Despawner` module is a utility script designed to handle the despawning of game objects.
- **[Dropoff](dropoff)** — The `Dropoff` module is responsible for managing the periodic dropping of random faction containers or vehicles by helicopters.
- **[Emplaced](emplaced)** — The `Emplaced` module manages the behavior of emplaced weapons, specifically handling events related to a player entering and exiting a gunner seat.
- **[FactionZone](factionzone)** — The `FactionZone` module adds a colored line region to the radar and PDA map for specific faction zones.
- **[FoodCart](foodcart)** — The `FoodCart` module is a utility script designed to handle interactions with food carts in the game.
- **[Fountain](fountain)** — The `Fountain` module provides utility functions for querying and handling interactions with fountains in the game world.
- **[FriendlyGate](friendlygate)** — The `FriendlyGate` module manages the behavior of gates that open based on proximity to friendly or allied vehicles.
- **[Fueltank](fueltank)** — The `Fueltank` module manages the visual effects associated with a fuel tank object.
- **[Goal](goal)** — The `Goal` module is responsible for handling the soccer easter egg in the game.
- **[Gurpodium](gurpodium)** — The `Gurpodium` module manages the in-vehicle speech cues for a specific type of vehicle.
- **[Hackybench](hackybench)** — The `Hackybench` module is a specialized version of the `Bench` module.
- **[Heavymg](heavymg)** — The `Heavymg` module is responsible for handling the removal of heavy machine gun (HMG) objects when they are dropped by a human player.
- **[Hero](hero)** — The `Hero` module is responsible for managing player-specific systems such as health regeneration, survival mode, inventory setup, and transfer system.
- **[HijackContractManager](hijackcontractmanager)** — The `HijackContractManager` module is responsible for managing active hijack contracts in the game.
- **[HomingMissile](homingmissile)** — The `HomingMissile` module represents a homing missile in the game world.
- **[IslandFortress](islandfortress)** — The `IslandFortress` module manages the collapse and destruction of an island fortress.
- **[Jammer](jammer)** — The `Jammer` module represents a context-toggle GPS jammer in the game.
- **[Laptop](laptop)** — The `Laptop` module represents a fixed support-munition pickup in the game world.
- **[LifestyleOilLif001Table](lifestyle_oillif001_table)** — The `LifestyleOilLif001Table` module is responsible for initializing and managing the state of a specific vehicle asset model named "OilLif001 Table".
- **[LivingWorldProp](livingworldprop)** — The `LivingWorldProp` module is designed to handle interactions with living world props that can be used by players.
- **[MaterialAnimation_LargeCanopy01](materialanimation_largecanopy01)** — The `MaterialAnimation_LargeCanopy01` module is responsible for triggering a material animation on a large canopy object when it enters the "CollapseFireState" state.
- **[MaterialAnimation_LargeCanopy02](materialanimation_largecanopy02)** — The `MaterialAnimation_LargeCanopy02` module is responsible for playing material animations on large canopy objects when they enter specific states, such as "CollapseFireState" or "CollapseState".
- **[MaterialAnimation_TreePlaza02](materialanimation_treeplaza02)** — The `MaterialAnimation_TreePlaza02` module is responsible for triggering material animations on a specific object in the game world.
- **[Mine](mine)** — The `Mine` module represents a mine object in the game.
- **[Monument](monument)** — The `Monument` module is a placeholder or stub for handling interactions with monument structures in the game world.
- **[MrxShootingGallery](mrxshootinggallery)** — The `MrxShootingGallery` module manages a shooting gallery mission scenario.
- **[Munitions](munitions)** — The `Munitions` module manages the behavior and interactions of munition objects in the game world.
- **[Oilrig](oilrig)** — The `Oilrig` module manages the destruction sequence of oil rig objects in the game.
- **[Outhouse](outhouse)** — The `Outhouse` module is a utility script that defines the behavior for using an outhouse in the game world.
- **[outpost](outpost)** — The `outpost` module manages the behavior and state of outposts in the game world.
- **[Paradrop](paradrop)** — The `Paradrop` module is responsible for deploying a squad of paratroopers via an aircraft.
- **[ParadropLocation](paradroplocation)** — The `ParadropLocation` module is responsible for triggering a paradrop plane when activated.
- **[Paratrooper](paratrooper)** — The `Paratrooper` module manages the behavior of paratroopers in the game.
- **[Pmcgate](pmcgate)** — The `Pmcgate` module is responsible for managing the activation and opening of PMC gates in the game world.
- **[ProximityMine](proximitymine)** — The `ProximityMine` module is responsible for handling the behavior of proximity mines in the game.
- **[Repairpad](repairpad)** — The `Repairpad` module manages the activation and deactivation of repair pads in the game world.
- **[ShootingGalleryTarget](shootinggallerytarget)** — The `ShootingGalleryTarget` module is responsible for handling state changes in shooting gallery targets.
- **[Soldier](soldier)** — The `Soldier` module handles the death logic for soldier AI entities in the game.
- **[SpyHunter](spyhunter)** — The `SpyHunter` module manages the behavior of the Spy Hunter vehicle in the game.
- **[Telephone](telephone)** — The `Telephone` module provides functionality for handling telephone interactions in the game.
- **[Treetrunk](treetrunk)** — The `Treetrunk` module is responsible for handling the state changes of tree trunks in the game.
- **[Treetrunkpalm](treetrunkpalm)** — The `Treetrunkpalm` module is responsible for playing material animations on palm tree trunks when they transition to specific fire-related states.
